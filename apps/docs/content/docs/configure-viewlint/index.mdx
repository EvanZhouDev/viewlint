ViewLint is configured with a `viewlint.config.ts` file.

If you used the quick start, you already have a working config. This section explains how it works and how to customize it.

## The mental model

ViewLint doesn’t lint “projects” or “files”. It lints **Targets**.

A Target is:

- a **View**: how to open the page and get it into the right state
- **option layers**: settings like `baseURL`, viewport, storage state, etc.
- an optional **scope**: which part of the page to lint

On the CLI, passing a URL is the easy path: ViewLint turns it into a Target by setting `baseURL`.
