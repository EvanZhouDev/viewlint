Most people use the CLI.

Use the TypeScript API when you want to:

- run ViewLint from a script
- build a custom integration
- lint custom Targets programmatically

## `ViewLint`

```ts
import { ViewLint } from "viewlint";
```

### `new ViewLint(options?)`

The constructor accepts an `options` object:

- `overrideConfigFile?: string` - use a specific config file path
- `baseConfig?: Config | Config[]` - applied before the discovered config file
- `overrideConfig?: Config | Config[]` - applied after everything else
- `plugins?: Record<string, Plugin>` - extra plugins (in addition to config)

## Lint targets

```ts
import { ViewLint } from "viewlint";
import rules from "@viewlint/rules";
import { defaultView, defineConfig } from "viewlint/config";

const viewlint = new ViewLint({
  baseConfig: defineConfig({
    plugins: { rules },
    extends: ["rules/recommended"],
  }),
});

const results = await viewlint.lintTargets([
  {
    view: defaultView,
    options: {
      context: {
        baseURL: "https://example.com",
      },
    },
  },
]);
```

In practice, most projects create Targets via config and the CLI.

Related: [Targets (mental model)](/docs/configure-viewlint)

## Format results

ViewLint ships with:

- `stylish`
- `json`

```ts
const formatter = await viewlint.loadFormatter("stylish");
const output = await formatter.format(results);
process.stdout.write(output);
```
